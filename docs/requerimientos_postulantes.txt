 Operación  Funcionalidad  Requerida  
Lógica  de  Negocio  
C  (Create)  Alta  manual  de  perfiles  espontáneos.  
Permite  al  reclutador  ingresar  manualmente  un  candidato  que  haya  llegado  por  canales  alternativos  (ej.  email  directo),  replicando  el  modelo  de  datos  de  la  Landing  Page.  
R  (Read)  Listado  y  previsualización.  
Tabla  o  grilla  de  tarjetas  ( Grid  View )  que  consuma  la  API  del  backend,  mostrando  el  volumen  total  de  postulantes  espontáneos  ordenados  por  fecha  de  creación  descendente  (createdAt).  
U  (Update)  Actualización  de  estado  y  datos.  
Funcionalidad  crítica  para  modificar  el  campo  estado_revision (ej.  de  "pendiente"  a  "revisado"  o  "descartado").  Permite  también  corregir  errores  tipográficos  en  el  correo  o  perfil  de  LinkedIn.  
D  (Delete)  Eliminación  de  registros  (Hard/Soft).  
Opción  para  descartar  permanentemente  un  registro  de  Firestore.  En  términos  de  arquitectura  de  backend,  esto  requerirá  asegurarse  de  que  el  PDF  asociado  en  Storage  también  sea  removido  para  no  dejar  archivos  huérfanos.
